# バリュー・プロポジション・キャンバス

## プロダクト名: LINE順番待ち

---

## 顧客プロファイル

### Customer Jobs（顧客のタスク）

1. **店舗運営者**: 混雑時の行列整理と顧客呼び出しを効率的に行う
2. **店舗運営者**: 待機中の顧客情報を把握し、リピーター化につなげる
3. **来店顧客**: 待ち時間を有効活用しながら、順番を逃さず入店する
4. **来店顧客**: 混雑状況を事前に把握して来店タイミングを決める

### Pains（痛み・不満）

1. スタッフが行列整理・呼び出し業務に追われ、本来の接客ができない
2. 顧客が長時間待てず、来店前に離脱してしまう（機会損失）
3. 待合室の混雑で3密状態になり、顧客・スタッフの不安が増す
4. 待機中の顧客属性がわからず、パーソナライズした対応ができない
5. 紙の整理券では呼び出し時に顧客が見当たらないことがある

### Gains（期待する成果）

1. 待ち時間の有効活用による顧客満足度向上
2. 行列整理業務の自動化によるスタッフ負担軽減
3. 待機中の自然な会員登録導線でLINE友だち数増加
4. 来店データの蓄積によるセグメントマーケティングの実現
5. 混雑緩和による快適な店舗環境の実現

---

## バリューマップ

### Products & Services（提供するサービス）

1. **QRコード整理券発行**: スマホで簡単に順番取得
2. **リアルタイム順番確認**: あと何組かをいつでも確認
3. **LINE自動通知**: 順番が近づくとプッシュ通知
4. **会員登録導線**: 待機中にシームレスな会員登録
5. **セグメント配信連携**: 来店後の自動追客メッセージ

### Pain Relievers（痛みの解消方法）

| 痛み | 解消方法 |
|------|----------|
| スタッフの行列整理負担 | QRコードによる自動発券とLINE通知による自動呼び出し |
| 顧客の待機離脱 | リアルタイム順番確認で「あと何組」が見える安心感 |
| 待合室の3密 | 院外・店外待機を可能にし、LINE通知で呼び戻し |
| 顧客情報の把握困難 | LINE連携による顧客属性・来店履歴の自動収集 |
| 呼び出し時の顧客不在 | LINE通知とリマインダーで確実に顧客へ到達 |

### Gain Creators（成果の実現方法）

| 期待する成果 | 実現方法 |
|--------------|----------|
| 顧客満足度向上 | 自由な待機時間と正確な順番通知による体験改善 |
| スタッフ負担軽減 | 発券・呼び出しの完全自動化 |
| LINE友だち数増加 | 整理券取得時の自然なLINE友だち追加導線 |
| セグメントマーケティング | 来店データと連携したグロースパックの配信機能 |
| 快適な店舗環境 | 分散待機による混雑緩和 |
