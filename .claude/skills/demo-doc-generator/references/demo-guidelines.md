# デモ構築ガイドライン

## 概要

グロースパック for LINEのデモを構築するためのガイドライン。
営業時に効果的なデモを実現し、デモの量産を可能にする。

## デモの要件

1. PCのブラウザでスマホ表示ができる
2. いちいちスマホを取り出す必要がない
3. グロースパックシリーズのデモが簡単に量産できる

## デモ構成

必ず以下の2つを構築する:

1. **LINEミニアプリ（プロトタイプ）**: 実際のアプリ画面
2. **デモ環境**: LINEミニアプリをPCブラウザで見せるための環境

## デモレイアウト構造

デモ環境は2つの部分で構成:

```
┌─────────────────────────────────────────┐
│  左サイドバー    │    iPhoneプレビュー    │
│  ・タイトル      │    ┌─────────────┐    │
│  ・ステップ一覧   │    │ ステータスバー │    │
│  ・リセットボタン │    │             │    │
│  ・進捗バー      │    │  コンテンツ   │    │
│                 │    │             │    │
│                 │    │ ホームバー    │    │
│                 │    └─────────────┘    │
└─────────────────────────────────────────┘
```

### 左サイドバー（コントロールパネル）

- デモタイトルと説明
- ステップナビゲーション（現在ステップをハイライト）
- リセットボタン（最初のステップに戻る）
- 進捗バー（current/total表示）

### 右側エリア（iPhoneプレビュー）

- リアルなiPhoneフレーム（rounded-[50px]、12pxボーダー）
- ステータスバー（SVGアイコン付き）
- コンテンツエリア（スクロール可能）
- ホームインジケーターバー

## 状態シミュレーションガイドライン

デモをリアルに見せるための状態シミュレーション:

### 1. 自動更新パターン

**用途**: 待ち時間カウントダウン、リアルタイム更新

```tsx
useEffect(() => {
  const interval = setInterval(() => {
    setCount((prev) => Math.max(prev - 1, 0));
  }, 8000); // 8秒ごとに更新
  return () => clearInterval(interval);
}, []);
```

### 2. 手動リフレッシュパターン

**用途**: データ取得のシミュレーション

```tsx
const [isRefreshing, setIsRefreshing] = useState(false);

const handleRefresh = () => {
  setIsRefreshing(true);
  setTimeout(() => {
    // データ更新
    setIsRefreshing(false);
  }, 500);
};
```

### 3. 条件付きUI表示

**用途**: 状態に応じたアラート表示

```tsx
const isNearby = waitingCount <= 3;

{isNearby && (
  <AlertCard title="まもなく順番です！" />
)}
```

## アニメーションガイドライン

Framer Motionを使用した標準アニメーション:

| ユースケース | パターン | 推奨設定 |
|------------|---------|---------|
| ページ読み込み | フェードイン + スライドアップ | delay: 0〜0.3秒 |
| リスト表示 | スタッガード | delay: i * 0.1秒 |
| 数値変化 | AnimatePresence | mode="wait" |
| 成功/完了 | スケールイン + リップル | type: "spring" |
| 注目喚起 | パルス | repeat: Infinity |
| ボタン | タップ | scale: 0.98 |

## プロトタイプガイドラインの構成

プロダクト固有のガイドラインを定義:

- ブランドカラーの適用（例: JAMESクーポンはBEAMSカラー）
- 機能要件のみ（非機能要件は不要）
- MVP機能の明確化
- **demoSteps設定**（demo-builderで直接使用可能）
- **状態シミュレーション要件**
- **アニメーション要件**
